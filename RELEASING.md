# Releasing DaisyUI Snippets

This document is the operational playbook for shipping the current milestone.

## Current Release Target

- version: `0.2.2`
- release tag: `v0.2.2`
- release branch: `codex/phase-6-release-ops`

## Branch Status

The milestone work is already chained linearly:

- `codex/phase-1-foundation`
- `codex/phase-3-library-polish`
- `codex/phase-4-release-hardening`
- `codex/phase-5-publishing-prep`
- `codex/phase-6-release-ops`

Because each later branch was created from the previous one, the simplest release path is:

1. open one PR from `codex/phase-6-release-ops` into `main`
2. squash merge it
3. tag `main` as `v0.2.2`
4. let GitHub Actions build and attach the `.vsix`
5. publish the same version to the VS Code Marketplace

## PR Checklist

Title recommendation:

- `Release 0.2.2: curated DaisyUI snippets, release hardening, and publishing prep`

PR body should include:

- product summary
- major snippet UX upgrades
- release workflow additions
- how to test
- release impact: `minor`

Use [release-notes/v0.2.2.md](./release-notes/v0.2.2.md) as the base for the summary.
Use [release-notes/pr-v0.2.2.md](./release-notes/pr-v0.2.2.md) for a ready-to-paste PR title and body.

## Pre-Merge Commands

Run these from the release branch:

```bash
npm install
npm run publish:precheck
code --install-extension bin/daisyui-vscode-snippets.vsix --force
code --show-versions --list-extensions | rg '^sjedt\\.daisyui-vscode-snippets@'
```

Expected result:

- `publish:precheck` passes
- VS Code reports `sjedt.daisyui-vscode-snippets@0.2.2`

## Merge and Tag

After the PR is approved:

```bash
git checkout main
git pull --ff-only
git merge --ff-only origin/main
git tag v0.2.2
git push origin main
git push origin v0.2.2
```

If the PR is squash-merged in GitHub first, refresh `main` locally before tagging:

```bash
git checkout main
git pull --ff-only origin main
git tag v0.2.2
git push origin v0.2.2
```

## GitHub Release

The tag triggers [.github/workflows/release.yml](./.github/workflows/release.yml).

Expected outcome:

- GitHub Actions runs the release gate
- `bin/daisyui-vscode-snippets.vsix` is uploaded
- a GitHub release is created automatically for `v0.2.2`

Paste the contents of [release-notes/v0.2.2.md](./release-notes/v0.2.2.md) into the release body if you want a more curated note than autogenerated release notes.

## Marketplace Publish

Authenticate once if needed:

```bash
npx @vscode/vsce login sjedt
```

Publish from `main` after the tag and release artifact exist:

```bash
git checkout main
git pull --ff-only origin main
npm install
npm run publish:precheck
npm run publish:vsce
```

## Post-Release Checks

After publishing:

1. confirm the Marketplace shows version `0.2.2`
2. install from Marketplace on a clean VS Code profile if possible
3. smoke-test:
   - `!d`
   - `d-alert`
   - `d-btn`
   - `d-card-image`
   - `d-dropdown`
   - `d-modal`
   - `d-navbar`
   - `d-table-zebra`
   - `d-tabs-lifted`
4. create the next branch from updated `main`
